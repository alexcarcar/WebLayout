Chapter 10 Self Test (p 372)

  1. Why does Java define both byte and character streams?

    • Byte streams provide a convenient means for handling input and output of bytes. They are used, for example, when reading or writing binary data.
    • Character streams are designed for handling the input and output of characters. Thy use Unicode and, therefore, can be internationalized.

  2. Even though console input and output is text-based, why does Java still use byte streams for this purpose?

    • At the lowest level, all I/O is still byte-oriented. (The original version of Java defined only the byte stream.)

  3. Show how to open a file for reading bytes.

    • FileInputStream fin = new FileInputStream("input.txt");

  4. Show how to open a file for reading characters.

    • BufferedReader br = new BufferedReader(new FileReader("input.txt");

  5. Show how to open a file for random-access I/O.

    • RandomAccessFile raf = new RandomAccessFile("input.txt", "rw");
