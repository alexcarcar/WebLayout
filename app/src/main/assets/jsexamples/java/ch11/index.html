Chapter 11 Self Test (p 414)

  1. How does Java's multithreading capability enable you to write more efficient programs?

    • Multithreading lets you utilize the idle time that is present in most programs.

  2. Multithreading is supported by the Thread class and the Runnable interface.

  3. When creating a runnable object, why might you want to extend Thread rather than implement Runnable?

    • When extending it is possible to include the ability to create and start a thread using a factory method.

  4. Show how to use join() to wait for a thread object called myThrd to end.

    • myThrd.join();

  5. Show how to set a thread called myThread to three levels above normal priority.

    • myThread.setPriority(Thread.NORM_PRIORITY+3);

  6. What is the effect of adding the synchronized keyword to a method?

    • When the method is called, the calling thread enters the object's , which then locks the object.
    • While locked, no other thread can enter the method, or enter any other synchronized method defined by the object's class
    • When the thread returns from the method, the monitor unlocks the object, allowing it to be used by the next thread.

  7. The wait() and notify() methods are used to perform interthread communication.

  8. Change the TickTock class so that it actually keeps time. That is, have each tick take one half second, and each tock take one half second. Thus, each tick-tock will take one second.

    • TickTockDemo.java

  9. Why can't you use suspend(), resume(), and stop() for new programs?

    • These Thread methods were deprecated in Java 2, because they can create deadlocks.

  10. What method defined in Thread obtains the name of a thread?

    • thread.getName();

  11. What does isAlive() return?

    • The isAlive() method returns true if the thread upon which it is called is still running.
