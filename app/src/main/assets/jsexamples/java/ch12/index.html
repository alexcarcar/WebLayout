  5. Define boxing and autoboxing. How does autoboxing/unboxing affect these actions?

    • is the process by which a primitive type is automatically encapsulated (boxed) into its equivalent type wrapper whenever an object of that type is needed.
    • is the process by which a primitive type is encapsulated (boxed) into its equivalent type wrapper by an explicitly cast.

  7. In your own words, what does static import do?

    • Adds static methods into your class, as if they are part of your class, i.e.:
    • import static java.lang.Math.sqrt;
    • x = sqrt(9);

  8. What does this statment do?

    • import static java.lang.Integer.parseInt;
    •  When using , it is possible to refer to static members directly by their names, without having to qualify them with the name of their class:
    • int i = parseInt("3");

  9. Is static import designed for special-case situations, or is it good practice to bring all static members of all classes into view?

    • It should be used in special-case situations, when a static member is used repeatedly.
